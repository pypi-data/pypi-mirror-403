# Inspect a Tenant-Facing Web Surface

Objective:
This lab verifies your ability to inspect a basic Azure
tenant-facing web surface and understand exposure and traffic flow.

You are reviewing a common Azure public web application pattern
before any resources are created.

---
Requirements:
You must reason strictly from the provided architecture description.
No assumptions about unmentioned Azure services or configurations
are allowed.

---
Task Requirements:

Edit the local file surface-review.md and explain:

1. Which components are publicly exposed
2. Which components remain private
3. How user traffic flows from the internet to the application
4. Why this exposure is appropriate for a tenant-facing web app

---
Constraints:

Do NOT design a new architecture
Do NOT introduce additional services
Do NOT propose improvements
Evaluate only what is described
Work fully offline and locally
