# Distinguish errors from warnings

Objective:

- Learn to distinguish warning signals from error conditions.
- Avoid escalating non-failure signals.
- Practice basic operational judgment using alerts.

---
Requirements:

- A set of alert messages is provided.
- Alerts represent normal system behavior with intermittent risk signals.
- The task must be completed fully offline.

---
Task Requirements:

- Review the provided alert messages.
- Identify which alerts represent actual system failure.
- Explain why other alerts do not indicate failure.
- Write your response in `severity-assessment.md`.

---
Constraints:

- Do NOT run commands or scripts.
- Do NOT modify the alert data.
- Do NOT propose fixes or remediation steps.
- Do NOT treat all warnings as incidents.
- Base your reasoning strictly on alert severity and wording.
