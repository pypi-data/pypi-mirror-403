# Read and understand basic application logs

Objective:

- Practice reading and understanding basic application logs.
- Identify error events and extract key operational details.
- Build foundational log literacy required for incident response and troubleshooting.

---
Requirements:

- A log file containing application events is provided.
- The log file must be treated as read-only.
- The task must be completed without executing scripts or automation.

---
Task Requirements:

- Review the provided application log carefully.
- Identify the first error event that appears in the log.
- Determine the timestamp at which the error occurred.
- Identify the component associated with the error.
- Create a file named answer.txt.
- Write your findings in clear, complete sentences.
- Use two short sentences: one stating when the first error occurs, and one stating which component it is associated with.
- Base your response strictly on the provided log data.

---
Constraints:

- Do NOT modify the log file.
- Do NOT use scripts, automation, or parsing tools.
- Do NOT assume information that is not explicitly present.
- Validation is static and based only on the contents of answer.txt.
