# Observe basic system metrics

Objective:

- Learn to read basic system metrics without interpretation bias.
- Identify clear changes in system behavior over time.
- Practice describing metrics factually, without guessing causes.

---
Requirements:

- A metrics snapshot representing system behavior over time is provided.
- Metrics include usage values recorded at regular intervals.
- The task must be completed fully offline.

---
Task Requirements:

- Review the provided metrics data carefully.
- Describe what changes are visible in the metrics.
- Avoid explaining *why* the changes occurred.
- Record your observations in **metrics-observation.md**.

---
Constraints:

- Do NOT run commands or scripts.
- Do NOT modify the metrics data.
- Do NOT infer root causes.
- Do NOT suggest fixes or remediation.
- Focus only on what the metrics objectively show.
