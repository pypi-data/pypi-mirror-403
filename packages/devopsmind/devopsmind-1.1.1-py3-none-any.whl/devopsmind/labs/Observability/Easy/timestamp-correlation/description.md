# Correlate events using timestamps

Objective:

- Learn to correlate events using timestamps.
- Understand how timing helps establish relationships between events.
- Avoid assuming causation without temporal evidence.

---
Requirements:

- A log file with time-ordered entries is provided.
- Logs represent a short incident window from a single service.
- The task must be completed fully offline.

---
Task Requirements:

- Review the provided log entries carefully.
- Identify the sequence of events leading to the failure.
- Explain how timing supports your reasoning.
- Record your analysis in **event-correlation.md**.

---
Constraints:

- Do NOT run commands or scripts.
- Do NOT modify the log file.
- Do NOT invent causes not supported by timestamps.
- Do NOT propose fixes or remediation steps.
- Focus only on event order and timing.
