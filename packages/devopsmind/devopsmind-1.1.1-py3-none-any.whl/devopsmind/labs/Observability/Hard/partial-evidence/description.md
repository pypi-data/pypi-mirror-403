# Perform RCA with partial observability data

Objective:

- Practice root cause analysis when evidence is incomplete.
- Learn to state uncertainty without guessing.
- Produce a disciplined RCA based only on available data.

---
Requirements:

- Metrics and logs captured during an incident are provided.
- Some observability signals are missing.
- The system state is preloaded.
- The task must be completed fully offline.

---
Task Requirements:

- Review all available evidence.
- Identify what can be concluded with confidence.
- Explicitly state what cannot be concluded due to missing data.
- Produce a structured root cause analysis.
- Record your analysis in **rca.md**.

---
Constraints:

- Do NOT run commands or scripts.
- Do NOT modify evidence files.
- Do NOT invent data or causes.
- Do NOT present speculation as fact.
- Clear uncertainty is preferred over confident guessing.
