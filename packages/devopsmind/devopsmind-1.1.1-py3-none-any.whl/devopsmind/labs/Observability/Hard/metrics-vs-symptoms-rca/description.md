# Distinguish symptoms from root cause using metrics

Objective:

- Perform evidence-based reasoning using metrics.
- Separate visible symptoms from the underlying cause.
- Produce a structured root cause analysis.

---
Requirements:

- A metrics snapshot representing system behavior during an incident is provided.
- The system state is preloaded and represents a live environment.
- The task must be completed fully offline.

---
Task Requirements:

- Review the provided metrics carefully.
- Identify which metrics describe symptoms and which indicate the underlying issue.
- Produce a structured root cause analysis.
- Record your analysis in **rca.md**.

---
Constraints:

- Do NOT run commands or scripts.
- Do NOT modify evidence files.
- Do NOT speculate beyond the metrics provided.
- Do NOT focus only on symptoms.
- Base conclusions strictly on evidence.
