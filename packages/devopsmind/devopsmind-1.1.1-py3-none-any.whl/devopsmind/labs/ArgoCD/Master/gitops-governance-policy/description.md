# Define Argo CD GitOps Governance Policy

Objective:

Define **senior-level governance policy** for Argo CD and GitOps usage
that balances autonomy, safety, and scalability across teams.

At this level, engineers are responsible for establishing standards
that prevent systemic failure without slowing delivery.

---
Task Requirements:

You are provided with **read-only organizational evidence**:

Incident Evidence - 

- `data/incident-summary.log`

Existing Standards - 

- `data/existing-standards.md`

Organizational Constraints - 

- `data/governance-constraints.md`

Policy Template - 

- `gitops-policy.md`

Using only the provided materials, define a GitOps governance policy
that includes -

1. Scope of applicability
2. At least three governance rules
3. Justification for each rule
4. Consideration of organizational constraints

---
Constraints:

- Do NOT propose new tools or platforms
- Do NOT reference automation or enforcement mechanisms
- Focus on **policy and standards**, not implementation
- Policies must be understandable by all engineering teams
