这玩意用 docker 部署会有些麻烦 首先他只能执行容器里面的命令
    也就是说 部署完证书之后 没法直接 重启 nginx  

其次 他不能操作 宿主机的  /etc/nginx/ssl 目录 除非做映射 
但是如果做了目录映射 可能会有个小问题 就是 ssl 这个目录算是被容器霸占了  也不算特别大的问题吧 

还有个问题 就是多个证书的情况 默认的证书名字是 nginx 一般情况下一定要手动更改证书名字  

一定要 能够重启 nginx 才行 要不然 不能应用证书 就不能实现完全自动化了
