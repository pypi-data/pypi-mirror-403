# Pre-Task Checklist

> Complete this checklist before starting any development task.

---

## 1. Context Understanding

- [ ] Read `project/context.md` for business background
- [ ] Check `project/known-issues.md` for related issues
- [ ] Review `bug-prevention.md` for common pitfalls

---

## 2. Requirement Clarity

- [ ] Task goal is clearly understood
- [ ] Scope boundaries are defined
- [ ] Success criteria are established
- [ ] Edge cases are identified

**If unclear, ask before proceeding!**

---

## 3. Technical Preparation

- [ ] Reviewed relevant code sections
- [ ] Identified affected modules/files
- [ ] Considered potential impacts
- [ ] Planned incremental approach

---

## 4. Risk Assessment

| Risk Type | Check |
|-----------|-------|
| Breaking changes | [ ] Assessed |
| Performance impact | [ ] Considered |
| Security implications | [ ] Reviewed |
| Data integrity | [ ] Verified |

---

## 5. Testing Strategy

- [ ] Unit tests planned
- [ ] Integration tests considered
- [ ] Test data isolation prepared (`autotest_` prefix)

---

## 6. Documentation

- [ ] Code comments planned
- [ ] README updates identified
- [ ] API doc changes needed?

---

## Quick Reference

```
Before coding:
1. Understand → 2. Plan → 3. Verify plan → 4. Execute incrementally
```
