______________________________________________________________________

## Module Capabilities

- Enforces Strict-Transport-Security, Referrer-Policy, Permissions-Policy, and other recommended
  headers with sane defaults.
- Ships FastAPI middleware and NestJS service wrappers that attach headers automatically to every
  response.
- Emits structured health payloads highlighting the active header set and any disabled protections.
- Supports configuration via generated vendor payloads, snippets, and override contracts for
  enterprise deployments.
- Provides snippets, tests, and documentation so downstream projects can adopt the module with
  minimal tuning.

## Directory Layout

| Path               | Responsibility                                                              |
| ------------------ | --------------------------------------------------------------------------- |
| `module.yaml`      | Canonical metadata (version, compatibility, documentation map)              |
| `config/base.yaml` | Declarative spec that drives prompts and dependency resolution              |
| `generate.py`      | CLI/automation entry point for rendering vendor and project variants        |
| `frameworks/`      | Framework plugin implementations registered via `modules.shared.frameworks` |
| `overrides.py`     | Runtime opt-in hooks applied via environment variables                      |
| `docs/`            | Module docs referenced from `module.yaml` (usage/overview/changelog)        |
| `templates/`       | Base templates plus per-framework variants                                  |

______________________________________________________________________

## Generation Workflow

1. `generate.py` loads `module.yaml` and checks version drift with
   `modules.shared.versioning.ensure_version_consistency`.
1. Vendor artefacts are rendered from templates into `.rapidkit/vendor/...` for reproducible
   installs.
1. The requested framework plugin maps templates into project-relative paths.
1. Optional lifecycle hooks (`pre_generation_hook`, `post_generation_hook`) handle final
   adjustments.

## Runtime Customisation

| Mechanism                                   | Effect                                                                                                   |
| ------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `SecurityHeadersSettings` (FastAPI)         | Pydantic model enabling CSP strings, permissions policy directives, and additional headers.              |
| `SecurityHeadersService` `options` (NestJS) | Constructor accepts partial overrides mirroring the Python defaults.                                     |
| `SecurityHeadersOverrides`                  | Override contract analogue for patching vendor behaviour at runtime.                                     |
| Environment variable `RAPIDKIT_VENDOR_ROOT` | Points the runtime wrappers to a custom vendor artefact directory when deploying with immutable systems. |

Use the override mixin to intercept header construction or to append tenant-specific metadata
without editing generated files.
